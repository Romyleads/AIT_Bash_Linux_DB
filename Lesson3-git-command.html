1.2 Клонировать форк git clone git clone https://github.com/ваш_юзер/cohort60_test1.git Скачать форк-репозиторий на локальный компьютер
1.3 Добавить upstream git remote add upstream git remote add upstream https://github.com/original/cohort60_test1.git Привязать оригинальный репозиторий как upstream
1.4 Проверить статус git status git status Показать текущий статус — изменения, ветка и т.п.
1.5 Проверить удалённые git remote -v git remote -v Показать список подключённых репозиториев
1.6 Создать .gitignore создать файл echo node_modules/ > .gitignore Создать файл с базовыми исключениями
1.7 Закоммитить .gitignore git add .gitignore git add .gitignore && git commit -m 'Добавлен .gitignore' Добавить и сохранить файл в истории
2.1 Создать ветку git checkout -b feature/update-readme git checkout -b feature/update-readme Создать новую ветку и переключиться на неё
2.2 Изменить README.md внести строку открыть README.md в редакторе, внести изменения Редактирование README.md через VS Code или GUI
2.3 Сделать коммит git add README.md git add README.md && git commit -m 'Добавлена строка в README' Добавить и зафиксировать изменения
2.4 Проверить статус git status git status Убедиться, что всё сохранено
2.5 Отправить ветку git push -u origin feature/update-readme git push -u origin feature/update-readme Отправить ветку в свой репозиторий на GitHub
2.6 Перейти в main git checkout main git checkout main Вернуться в ветку main
2.7 Три файла и три коммита создать и коммитить по одному touch file1.txt && git add file1.txt && git commit -m 'file1' (повторить 3 раза) Создать 3 файла и для каждого — отдельный коммит
2.8 Просмотр истории git log --oneline --graph --all git log --oneline --graph --all Визуализированная история всех веток
2.9 Объединить коммиты git rebase -i HEAD~2 git rebase -i HEAD~2 Интерактивный rebase для объединения последних коммитов

4.2 Изменить README.md внести строку открыть README.md в редакторе, изменить Изменение строки в README.md через редактор

4.4 Изменить ту же строку в main изменить и закоммитить открыть README.md, изменить, git add + commit Вызвать конфликт вручную

4.6 Разрешить конфликт вручную отредактировать, git add, git commit Решить конфликт вручную и завершить слияние

5.3 Снова обновить main git checkout main && git pull upstream main git checkout main && git pull upstream main Актуализировать основную ветку
5.4 Rebase git rebase main git checkout feature/rebase-test && git rebase main Наложить изменения на обновлённый main
